## Metadata
**Name:** `kubernetes/workload_mounting_with_sensitive_os_directory`

**Id:** `a737be28-37d8-4bff-aa6d-1be8aa0a0015`

**Cloud Provider:** kubernetes

**Severity:** High

**Category:** Insecure Configurations

## Description
Workload is mounting a volume with sensitive OS Directory

#### Learn More

 - [Provider Reference](https://registry.terraform.io/providers/hashicorp/kubernetes/latest/docs/resources/pod#host_path)

## Non-Compliant Code Examples
```terraform
resource "kubernetes_pod" "test1" {
  metadata {
    name = "terraform-example1"
  }

  spec {
    container {
      image = "nginx:1.7.9"
      name  = "example"

      env {
        name  = "environment"
        value = "test"
      }

      port {
        container_port = 8080
      }

      liveness_probe {
        http_get {
          path = "/nginx_status"
          port = 80

          http_header {
            name  = "X-Custom-Header"
            value = "Awesome"
          }
        }

        initial_delay_seconds = 3
        period_seconds        = 3
      }

    }

    dns_config {
      nameservers = ["1.1.1.1", "8.8.8.8", "9.9.9.9"]
      searches    = ["example.com"]

      option {
        name  = "ndots"
        value = 1
      }

      option {
        name = "use-vc"
      }
    }

    volume {
      host_path {
        path = "/var/log"
      }
    }

    dns_policy = "None"
  }
}
```
